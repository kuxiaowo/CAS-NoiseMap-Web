# system_config.json 字段说明

这个文件保存**前端显示相关配置**，也就是地图页面怎么画、多久刷新一次、图例怎么显示。

---

## 顶层结构

```json
{
  "frontend": { ... }
}
```

### `frontend`
前端地图与页面显示配置。

字段说明：

- `poll_interval_ms`
  - 前端从后端拉取最新状态的频率，单位毫秒
  - 例如 `1000` 表示每 1 秒刷新一次页面数据
  - 这个值越小，页面看起来越“实时”，但浏览器和后端请求也会更多

- `scale`
  - 地图坐标缩放系数
  - 传感器的 `x/y` 会乘以这个值后再绘制到画布上
  - 如果点位整体太稀或太密，可以改它

- `point_radius`
  - 地图上圆点本体的半径
  - 越大越显眼

- `influence_radius`
  - 传感器热区影响半径
  - 表示一个点周围的颜色扩散范围
  - 越大，热区越宽

- `influence_opacity`
  - 热区透明度
  - 范围通常在 `0 ~ 1`
  - 越大越浓，越小越淡

- `map_image`
  - 地图背景图文件名或路径
  - 前端会加载它作为底图

- `map_anchor`
  - 地图锚点，用来决定地图图片相对于画布中心怎么对齐
  - 结构：

```json
"map_anchor": { "x": 800, "y": 780 }
```

  - 可以理解为：把地图图片中的哪个点，对准画布中心附近的位置

- `map_scale`
  - 地图背景图片本身的缩放倍数
  - 只影响底图大小，不直接改传感器坐标值

- `outside_color`
  - 地图外部背景色
  - 也就是底图没覆盖到的区域显示什么颜色

- `axis_color`
  - 中心坐标轴线颜色
  - 当前页面会画一条水平轴和一条垂直轴，用于辅助观察

- `show_labels`
  - 是否显示点位标签和数值
  - `true` 表示显示，`false` 表示隐藏

- `glow_intensity`
  - 热区发光强度系数
  - 越大，热区叠加效果越明显

- `grid_enabled`
  - 是否显示网格辅助线
  - `true` 显示网格，`false` 不显示

- `legend`
  - 图例配置数组
  - 决定右侧“颜色说明”里显示哪些档位文字

每一项结构类似：

```json
{ "label": "低噪音", "color": [0, 200, 0], "desc": "< 55" }
```

字段说明：

- `label`
  - 图例名称

- `color`
  - 图例颜色，格式 `[R, G, B]`

- `desc`
  - 图例说明文字
  - 一般写对应的分贝区间

## 调整建议

### 如果页面刷得太勤
改：
- `poll_interval_ms`

推荐先用：
- `1000`
- `1500`

### 如果点位在地图上偏得太厉害
优先检查：
- 传感器自己的 `x/y`

如果是**整张图整体不对位**，再调：
- `map_anchor`
- `map_scale`
- `scale`

### 如果热区太夸张或太弱
调：
- `influence_radius`
- `influence_opacity`
- `glow_intensity`

### 如果页面太乱
可以考虑：
- 把 `show_labels` 改成 `false`
- 把 `grid_enabled` 改成 `false`
- 适当减小 `point_radius`
