# sensors.json 字段说明

这个文件保存**传感器业务配置**，包括是否启用、地图坐标、上报频率、三点校准参数、颜色阈值。

---

## 顶层结构

```json
{
  "defaults": { ... },
  "thresholds": [ ... ],
  "sensors": { ... }
}
```

### `defaults`
默认值模板。

作用：当你新增一个传感器，或者某个传感器没显式填写某项时，会优先参考这里。

字段说明：

- `enabled`
  - 是否默认启用新传感器

- `label_prefix`
  - 新传感器默认标签前缀

- `report_interval_ms`
  - 默认上报频率，单位毫秒

- `x` / `y`
  - 默认地图坐标

- `calibration_points`
  - **三点校准数组**，固定建议写 3 组点
  - 每组格式：

```json
{ "db": 60.0, "raw_rms": 120.0 }
```

  - 含义是：在真实环境大约 `60 dB` 时，这个传感器测到的原始 `raw_rms` 大约是 `120.0`
  - 后端会按 `raw_rms` 从小到大排序，并在相邻点之间做**对数分段插值**
  - 这样比单参考点更稳，低声、中声、高声三个区间都更容易贴近实际

- `min_db`
  - 换算结果的最小钳制值

- `max_db`
  - 换算结果的最大钳制值

---

### `thresholds`
颜色阈值配置。

每一项结构类似：

```json
{ "max": 55.0, "rgb": [0, 200, 0], "name": "green" }
```

字段说明：

- `max`，当前阈值上限
- `rgb`，前端显示颜色
- `name`，这一档的内部名称

注意：
- 阈值按顺序判断
- 最后一档通常设成一个很大的值做兜底

---

### `sensors`
每个传感器的独立配置，key 是传感器 ID。

单个传感器对象常用字段：

- `id`
- `enabled`
- `label`
- `x`
- `y`
- `report_interval_ms`
- `calibration_points`
- `min_db`
- `max_db`

示例：

```json
"3": {
  "id": 3,
  "enabled": true,
  "label": "传感器 3",
  "x": 380.0,
  "y": -20.0,
  "report_interval_ms": 1000,
  "calibration_points": [
    { "db": 50.0, "raw_rms": 8.6 },
    { "db": 65.0, "raw_rms": 18.2 },
    { "db": 80.0, "raw_rms": 42.0 }
  ],
  "min_db": 30.0,
  "max_db": 130.0
}
```

## 校准建议

### 怎么测这 3 个点
建议至少选 3 个明显不同的声压点，例如：

- 安静但非静音环境
- 中等说话/设备运行环境
- 更高一点的稳定噪音环境

每个点记录两项：

1. 真实参考分贝 `db`
2. 传感器对应的 `raw_rms`

### 怎么填更稳
- 3 个点尽量覆盖你最关心的工作区间
- `raw_rms` 要从小到大递增
- 不要让 3 个点挤得太近
- 如果某台设备整体偏高/偏低，优先重测这 3 个点，不要硬调阈值

### 如果你只是想挪地图位置
只改：
- `x`
- `y`

不用动校准项。

## 备注

现在网页端配置面板支持**直接修改并保存**。

你也可以直接编辑：

- `sensors.json`

如果是网页端修改，保存后会立即写回配置文件。
如果是手动改文件，改完后刷新页面，或重启后端以确认新配置生效。
