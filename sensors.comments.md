# sensors.json 字段说明

这个文件保存**传感器业务配置**，包括是否启用、地图坐标、上报频率、校准参数、颜色阈值。

---

## 顶层结构

```json
{
  "defaults": { ... },
  "thresholds": [ ... ],
  "sensors": { ... }
}
```

### `defaults`
默认值模板。

作用：当你新增一个传感器，或者某个传感器没显式填写某项时，会优先参考这里。

字段说明：

- `enabled`
  - 是否默认启用新传感器
  - `true` 表示默认启用，`false` 表示默认禁用

- `label_prefix`
  - 新传感器默认标签前缀
  - 例如设成 `"传感器 "`，ID=3 时默认会生成 `传感器 3`

- `report_interval_ms`
  - 默认上报频率，单位毫秒
  - 例如 `1000` 表示每 1 秒上报一次
  - 这个值越小，刷新越快，但设备和网络压力也更高

- `x`
  - 默认地图 X 坐标
  - 新传感器未设置坐标时使用

- `y`
  - 默认地图 Y 坐标
  - 新传感器未设置坐标时使用

- `reference_raw_rms`
  - 校准参考原始 RMS
  - 表示“某个参考声压条件下，设备测到的原始 RMS 大约是多少”
  - 后端会用它把原始 RMS 换算成 dB

- `reference_db`
  - 与 `reference_raw_rms` 对应的参考分贝值
  - 例如你知道某设备在 60dB 环境下测得 RMS 大约 20，就可以写：
    - `reference_raw_rms = 20`
    - `reference_db = 60`

- `noise_floor_raw_rms`
  - 设备底噪对应的原始 RMS
  - 用于抵消麦克风和电路本身的背景噪声
  - 如果设太低，安静环境下读数可能偏高
  - 如果设太高，低声压会被压扁

- `min_db`
  - 换算结果的最小钳制值
  - 防止过低读数出现不合理的负值或极小值

- `max_db`
  - 换算结果的最大钳制值
  - 防止异常尖峰把显示拉到离谱范围

---

### `thresholds`
颜色阈值配置。

作用：决定后端把噪音值显示成什么颜色、什么等级。

每一项结构类似：

```json
{ "max": 55.0, "rgb": [0, 200, 0], "name": "green" }
```

字段说明：

- `max`
  - 当前阈值上限
  - 当噪音值 `< max` 时，就命中这一档

- `rgb`
  - 前端显示颜色，格式是 `[R, G, B]`

- `name`
  - 这一档的内部名称
  - 后端会把它一起返回给前端

注意：
- 阈值是**按顺序判断**的
- 所以后面的阈值应当比前面更高
- 最后一档通常设成一个非常大的值，兜底所有更高噪音

---

### `sensors`
每个传感器的独立配置。

结构是以传感器 ID 作为 key：

```json
"sensors": {
  "1": { ... },
  "2": { ... }
}
```

每个传感器对象字段说明：

- `id`
  - 传感器 ID
  - 应与 key 对应，也应与设备实际上报的 ID 一致

- `enabled`
  - 是否启用该传感器
  - `false` 时：
    - 地图不显示
    - 后端仍可保留配置
    - 设备下次同步配置时也会收到禁用状态

- `label`
  - 地图和管理界面中显示的名称

- `x`
  - 地图上的 X 坐标

- `y`
  - 地图上的 Y 坐标

- `report_interval_ms`
  - 希望设备使用的上报频率，单位毫秒
  - 设备会定期从后端同步这个值

- `reference_raw_rms`
  - 该设备自己的校准参考原始 RMS
  - 不同麦克风灵敏度不同，所以每个设备最好单独调

- `reference_db`
  - 上面那个参考 RMS 对应的参考 dB

- `noise_floor_raw_rms`
  - 该设备的底噪原始 RMS

- `min_db`
  - 该设备换算时允许的最小 dB

- `max_db`
  - 该设备换算时允许的最大 dB

---

## 调整建议

### 如果你想让地图刷新更快
优先改：
- 具体传感器的 `report_interval_ms`

但不要一口气拉太低，建议先从：
- `1000`
- `700`
- `500`

逐步试。

### 如果你觉得某个传感器偏高或偏低
优先调：
- `reference_raw_rms`
- `noise_floor_raw_rms`

### 如果你只是想挪地图位置
只改：
- `x`
- `y`

不用动校准项。
